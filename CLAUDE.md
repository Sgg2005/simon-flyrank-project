# Project conventions

## Stack
- Frontend: HTML, CSS, Javascript & React
- Tools: Claude Code, GitHub

## Conventions
- Commit messages will follow the Conventional Commits format (examples: `feat:`, `fix:`, `docs:`, `chore:`)
- I'll keep my code neat and easy to understand
- I'll use the correct HTML tags for each part of the page
- I'll only use React when something needs to be interactive or reused in multiple places

## Rules I learnt in the prompting drill
- Always ask the AI to write and run tests in the same prompt, not as a seperate follow up
- once the AI creates a new component, check if it has been added and shown do not always
assume it does these things automatically
- A vague, one-sentence prompt can still look polished, but that doesn't mean it's actually correct only real tests confirm that.
